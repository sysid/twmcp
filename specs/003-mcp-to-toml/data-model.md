# Data Model: MCP Config to TOML Extractor

**Date**: 2026-02-18
**Feature**: 003-mcp-to-toml

## Input: MCP JSON Config

Three structural variants, all containing the same server entry shape:

### Variant A: Claude Desktop / Copilot CLI

```json
{
  "mcpServers": {
    "<server-name>": { <server-entry> }
  }
}
```

### Variant B: VS Code

```json
{
  "mcp": {
    "servers": {
      "<server-name>": { <server-entry> }
    }
  }
}
```

### Variant C: Flat / IntelliJ

```json
{
  "servers": {
    "<server-name>": { <server-entry> }
  }
}
```

### Server Entry Shape

```
server-entry := {
  "command"?: string,           # executable path/name (stdio servers)
  "args"?:    [string, ...],    # command arguments
  "type"?:    string,           # "stdio" | "local" | "http" | "sse" (default: "stdio")
  "env"?:     { string: string }, # environment variables
  "url"?:     string,           # endpoint URL (http/sse servers)
  "headers"?: { string: string }, # HTTP headers (http/sse servers)
  "tools"?:   [string, ...],    # tool filter
  ...<unknown>                   # any other keys → preserved as comments
}
```

### Known Properties (allowlist)

| Property | Type | Notes |
|----------|------|-------|
| `command` | string | Required for stdio servers |
| `args` | string[] | Command arguments |
| `type` | string | Transport type, default "stdio" |
| `env` | dict[str, str] | Environment variables |
| `url` | string | Required for http/sse servers |
| `headers` | dict[str, str] | HTTP headers |
| `tools` | string[] | Tool filter list |

## Output: Canonical TOML

Matches the existing `config.py:CanonicalConfig` / `config.py:Server` structure:

```toml
# Generated by: twmcp extract <source-file>
# Secret placeholders (set these environment variables):
#   GITHUB_TOKEN, API_SECRET

[servers.<name>]
command = "<value>"
args = ["<arg1>", "<arg2>"]
type = "<canonical-type>"
# unknown: disabled = true

[servers.<name>.env]
GITHUB_TOKEN = "${GITHUB_TOKEN}"
PLAIN_VALUE = "literal-value"

[servers.<name>.headers]
X-Custom-Header = "${X_CUSTOM_HEADER}"
```

## Transformations

### Type Normalization

| Input value | Canonical output |
|-------------|-----------------|
| `"local"` | `"stdio"` |
| `"stdio"` | `"stdio"` |
| `"http"` | `"http"` |
| `"sse"` | `"sse"` |
| other | preserved as-is + `# unrecognized type` comment |

### Secret Placeholder Substitution

Applied to values in `env` and `headers` dicts.

**Trigger**: Key name ends with `_TOKEN`, `_KEY`, `_SECRET`, `_PASSWORD`, or `_CREDENTIALS` (case-insensitive suffix match).

**Transform**: `KEY_NAME = "literal-value"` → `KEY_NAME = "${KEY_NAME}"`

**Tracking**: All substituted variable names are collected and listed in a header comment block.

### Unknown Property Handling

Any JSON key in a server entry not in the known properties allowlist:

`"disabled": true` → `# unknown: disabled = true`
`"customField": "value"` → `# unknown: customField = "value"`
