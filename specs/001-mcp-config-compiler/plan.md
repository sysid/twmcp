# Implementation Plan: MCP Config Compiler

**Branch**: `001-mcp-config-compiler` | **Date**: 2026-02-17 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-mcp-config-compiler/spec.md`

## Summary

Build a CLI tool (`twmcp`) that reads MCP server definitions from a
single canonical TOML config, resolves variable placeholders from
environment variables and dotenv files, and compiles agent-specific
JSON config files for Copilot CLI, IntelliJ Copilot, and Claude
Desktop — each with its own format, top-level key, and file location.

## Technical Context

**Language/Version**: Python 3.13+
**Primary Dependencies**: typer (CLI + Rich output), tomllib (stdlib)
**Storage**: File-based (TOML in, JSON out)
**Testing**: pytest (constitution: 85% coverage, markers for integration)
**Target Platform**: macOS (primary), Linux (secondary)
**Project Type**: single
**Performance Goals**: N/A (config compilation is instant)
**Constraints**: Minimal dependencies (constitution: prefer stdlib)
**Scale/Scope**: Single-user CLI tool, 3 agent targets initially

**Structure Decision**: Single project layout per constitution. All
source in `twmcp/`, all tests in `tests/`. No sub-packages needed —
5 modules at ~50-100 lines each is appropriate for this scope.

## Module Responsibilities

| Module | Responsibility | Dependencies |
|--------|---------------|--------------|
| `cli.py` | Typer app, command definitions, error display | config, agents, compiler |
| `config.py` | Load TOML, parse into CanonicalConfig model, validate | interpolate |
| `agents.py` | Agent profile registry, profile lookup | (none) |
| `compiler.py` | Transform canonical → agent-specific dict, write JSON | agents |
| `interpolate.py` | Resolve `${VAR:-default}` from env + dotenv | (none) |

Dependency direction: `cli → compiler → config → interpolate`
                                      `→ agents`
